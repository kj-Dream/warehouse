🔰 第一阶段：看懂"骨架"——一个请求怎么走完三层（3天）
📅 Day 1：跟踪"登录"请求的完整旅程
目标：理解 Controller → Service → Mapper 三层是怎么串起来的
顺序文件类型怎么看
1️⃣controller/LoginController.java阅读从第84行 @PostMapping("/login") 开始，看它怎么接请求、怎么验参数、怎么调 Service
2️⃣dto/LoginRequest.java阅读前端发来的JSON长什么样，这个类就是什么结构
3️⃣service/impl/LoginServiceImpl.java核心登录的 6 步流程：查用户→验密码→升级加密→查状态→查角色→存Session
4️⃣mapper/SysUserMapper.java阅读Mapper 接口长什么样，和 XML 怎么对应
5️⃣resources/dao/SysUserMapper.xml核心SQL 写在哪里，#{} 占位符是什么，resultMap 是干嘛的
6️⃣dto/LoginResponse.java了解为什么返回数据里没有 password 字段（数据脱敏）
7️⃣dto/Result.java泛型所有接口统一返回格式：{code, msg, data} ——这是怎么实现的
💡 学习方法：左右分屏打开 Controller 和 ServiceImpl，逐行对照看。拿纸笔画出请求流程图。

✅ 学好的标准：能口述"浏览器发来 username+password，最终怎么变成 SQL 查询的"

📅 Day 2：看懂"注册"流程 + 密码加密
目标：理解 BCrypt 加密和 Session 机制
顺序文件类型怎么看
1️⃣util/PasswordUtil.java核心BCrypt 加密的原理：为什么同一个密码两次加密结果不一样？"盐值"是什么？
2️⃣dto/RegisterRequest.java阅读注册比登录多哪些字段
3️⃣service/impl/LoginServiceImpl.java核心看 register() 方法：为什么注册时 status=0（待审核），管理员创建时 status=1（直接启用）
4️⃣mapper/SysUserMapper.xml关键看 INSERT 语句：COALESCE 函数的作用（如果Java没传值就用默认值）
💡 核心概念：Session 是服务端存储的"用户身份证"，登录时存进去，之后每次请求都能取出当前用户。

✅ 学好的标准：能解释为什么密码不能明文存、注册后为什么不能直接登录（需审核）

📅 Day 3：项目的"骨架"——配置与工具
目标：理解项目是怎么跑起来的
顺序文件类型怎么看
1️⃣WarehouseApplication.java阅读Spring Boot 的入口，@SpringBootApplication 做了什么
2️⃣resources/application.yaml核心数据库连接配置、端口号、MyBatis XML 路径
3️⃣config/WebConfig.java核心CORS 跨域配置——为什么前端5173端口能访问后端8080端口
4️⃣pom.xml了解项目依赖了哪些库（Spring Boot、MyBatis、MySQL、Lombok）
⚠️ 关键对比：打开 application.yaml，看 MySQL 连接字符串，再打开 pom.xml 看 mysql-connector-j 依赖，理解"驱动"的作用。

✅ 学好的标准：能新建一个空的 Spring Boot 项目，自己配好数据库连接和 CORS

🔧 第二阶段：搞懂"积木"——每一层怎么写（4天）
📅 Day 4：MyBatis 动态 SQL —— 最核心的技能
目标：学会写带条件的 SQL，理解 MyBatis 的 XML 语法
顺序文件类型怎么看
1️⃣resources/dao/SysUserMapper.xml核心重点看：<sql> 复用片段、<if> 条件判断、<foreach> 批量操作
2️⃣resources/dao/SysRoleMapper.xml阅读对比学习：简单查询 vs 复杂分页查询的 SQL 写法
3️⃣resources/dao/InventoryMapper.xml高级看 JOIN 多表联查和 ON DUPLICATE KEY UPDATE 的用法
💡 核心概念：#{xxx} 是参数占位符（防 SQL 注入），${xxx} 是字符串替换（有注入风险，尽量不用）

✅ 学好的标准：能写出"根据多个可选条件筛选 + 分页"的完整 SQL 语句

📅 Day 5：Service 层 —— 业务逻辑怎么写
目标：理解事务、异常处理、业务校验的写法
顺序文件类型怎么看
1️⃣service/impl/UserServiceImpl.java核心完整的增删改查业务：分页计算、密码加密、重复校验、批量操作
2️⃣service/impl/RoleServiceImpl.java核心重点看：@Transactional 事务 + "先删后插"更新权限的策略
3️⃣service/impl/StockServiceImpl.java高级出入库审核联动库存：为什么审核通过后才改库存？事务保证一致性
⚠️ 重要理解：@Transactional 的作用是"要么全成功，要么全失败"——比如创建角色时，角色插入了但权限没插入，事务会让角色也回滚掉。

✅ 学好的标准：能解释 @Transactional 什么时候用，为什么角色更新要用"先删后插"

🚀 第三阶段：打通"经脉"——核心业务流程（4天）
📅 Day 8：RBAC 权限体系
目标：理解权限控制最核心的四张表关系
顺序文件类型怎么看
1️⃣数据库四张表核心sys_user → sys_role → sys_role_menu → sys_menu
用户表存 role_id，角色菜单关联表存 role_id+menu_id
2️⃣service/impl/SysMenuServiceImpl.java核心两个关键方法：
① 平铺列表→树形结构（buildTree）
② 按角色过滤菜单（getMenuTreeByRoleId）
3️⃣controller/MenuController.java阅读从 Session 取当前用户角色，返回过滤后的菜单树
💡 核心模型：RBAC = Role-Based Access Control。管理员创建角色→给角色分配菜单→把角色分配给用户。用户登录后根据角色看不同菜单。

✅ 学好的标准：能画出四张表的 ER 图，解释"怎么实现 admin 看到全部菜单、仓管员只看到业务菜单"

📅 Day 9：出入库业务流 + 库存联动
目标：理解业务流程中数据如何联动变更
顺序文件类型怎么看
1️⃣service/impl/StockServiceImpl.java核心看 createStockIn() 和 updateStockInStatus()：审核通过后自动调 inventoryMapper.addStock()
2️⃣resources/dao/InventoryMapper.xml核心addStock 用了 ON DUPLICATE KEY UPDATE——有记录就累加，没记录就插入
3️⃣controller/StockController.java阅读看审核接口：/audit/stock-in/{id}/approve 和 reject，理解审核流程
⚠️ 关键逻辑：为什么审核通过后才改库存？因为提交不等于审核通过，驳回的单据不应该影响库存。这叫"事务的最终一致性"。

✅ 学好的标准：能口述"采购入库 → 保存草稿 → 提交 → 管理员审核通过 → 库存自动增加"的完整链路
