🔐 LoginController

所有请求以 /api/auth 开头，@RequestMapping("/api/auth")。
POST /api/auth/login
🌐 浏览器
发送请求{"username":"admin","password":"123456","captcha":8}
🎯 Controller
① 参数校验username、password 非空
⬇
② 验证码校验session.getAttribute("captchaAnswer")
比对用户输入的 captcha
⬇
③ 调 ServiceloginService.login(request, session)
⚙️ Service
① 查用户sysUserMapper.findByUsername("admin")
⬇
② BCrypt 比对PasswordUtil.matches("123456", "$2a$10$...")
如果数据库是明文，自动升级为BCrypt
⬇
③ 检查 statususer.getStatus()==1 才能登录
⬇
④ 查角色sysRoleMapper.findById(roleId)
⬇
⑤ 组装返回值LoginResponse（不含密码）
⬇
⑥ 存Sessionsession.setAttribute("user", response)

⬇

⬇

PUT /api/audit/stock-in/{id}/approve（审核通过→自动加库存）
🎯 Controller
调Service
stockService.updateStockInStatus(id, "approved")
⚙️ Service
@Transactional
① 检查入库单存在
⬇
② 更新状态stockInMapper.updateStatus(id, "approved")
⬇
③ 查询明细stockInDetailMapper.findByStockInId(id)

📁 Mapper
InventoryMapper.xml
INSERT INTO inventory (product_id,warehouse_id,location_id,quantity,version) VALUES (?,?,?,?,1) ON DUPLICATE KEY UPDATE quantity = quantity + ?, version = version + 1
没有记录→插入，有记录→累加！
🗄️ 返回
{code:200, msg:"入库单已审核通过，库存已更新"}
PUT /api/audit/stock-out/{id}/approve（审核通过→自动减库存，库存不足则报错）
⚙️ Service 审核出库
@Transactional
① 更新状态stockOutMapper.updateStatus
⬇
② 查明细 → 扣库存inventoryMapper.subtractStock
如果返回rows==0 → 库存不足！
③ 抛异常"商品库存不足"→ 事务回滚
📁 Mapper
UPDATE inventory SET quantity = quantity - ?, version = version + 1 WHERE product_id = ? AND warehouse_id = ? AND location_id = ? AND quantity >= ?
WHERE quantity >= ? 防止扣成负数！

🎯 Controller
接收驳回理由（可选）

⚙️ Service

📌 后端架构总结
🔹 9 个 Controller / 67 个 API 接口 / 14 张数据库表
🔹 每个请求的路径：前端 → Controller(接参+校验) → Service(业务+事务) → Mapper(SQL) → MySQL → 反向 JSON 返回
🔹 关键设计：Result统一响应、RBAC动态菜单、BCrypt密码、审核联动库存、@Transactional事务
🔹 技术栈：Spring Boot 3.5.15 + MyBatis 3.0.5 + MySQL 8.0 + Vue 3 + Element Plus 2.7
